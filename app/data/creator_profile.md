## Principais projetos técnicos

### Lylla — Organismo Cognitivo Experimental

Lylla é um organismo digital experimental de Vida Artificial desenvolvido em Python. O projeto investiga uma arquitetura cognitiva bottom-up, construída a partir de mecanismos como percepção, memória persistente, associação, hipóteses, avaliação, abstração, linguagem, sonho, governança semântica e aprendizado procedural.

A proposta da Lylla não é ser apenas uma interface conversacional. O projeto busca explorar como estruturas cognitivas mais complexas podem surgir da integração progressiva de módulos menores. Ele trabalha com ciclos contínuos, memória semântica, relações estruturadas, controle epistêmico, rastreabilidade e mecanismos de reorganização interna.

A Lylla representa o interesse de Alexandre por inteligência artificial além do uso tradicional de prompts: sistemas com memória, evolução, controle de qualidade, explicabilidade e experimentação cognitiva.

Para mais detalhes técnicos, consulte o arquivo específico sobre a Lylla.

### Legacy of Ashes: Reborn — RPG Web de Fantasia Sombria

Legacy of Ashes: Reborn é um RPG de fantasia sombria para navegador, criado por Alexandre. O jogo coloca o jogador em um mundo devastado pela Calamidade, partindo da Vila das Cinzas para explorar regiões perigosas, enfrentar criaturas, coletar equipamentos, evoluir personagens e descobrir fragmentos da história do mundo.

O projeto combina criação de personagem, combate por turnos, progressão, inventário, equipamentos, economia, missões, contratos, desafios, mercado rotativo, bênçãos, save online, chat, amigos, ranking, códigos promocionais e expedições cooperativas para dois jogadores.

Legacy of Ashes demonstra a capacidade de Alexandre de construir um produto digital complexo, envolvendo frontend, backend, banco de dados, assets, sistemas de jogo, experiência mobile, deploy e persistência online.

Para mais detalhes técnicos e de gameplay, consulte o arquivo específico sobre Legacy of Ashes.

### JackFin — Plataforma de Trading Algorítmico

JackFin é uma plataforma de trading algorítmico de criptomoedas. Em sua versão V4, o projeto foi estruturado com arquitetura orientada a eventos, Event Sourcing, CQRS, TypeScript estrito, PostgreSQL, TimescaleDB, Redis, Fastify, React, integração com exchange, inteligência de mercado e backtesting.

O sistema foi desenhado para ser auditável, determinístico e extensível. A ideia central é que bots sejam definidos por configuração, não por novo código. Cada mudança importante gera eventos imutáveis, permitindo reconstrução de estado, replay e análise de decisões.

O JackFin demonstra o interesse de Alexandre por automação financeira, engenharia de estratégia, dados de mercado, arquitetura robusta, simulação, análise de risco e sistemas orientados por eventos.

Para mais detalhes técnicos, consulte o arquivo específico sobre JackFin.

### Projeto Andromeda — Plataforma Investigativa Web

Projeto Andromeda é uma plataforma web investigativa criada com HTML, CSS, JavaScript e Particle.js. O projeto simula arquivos criminais, classificação de risco, terminal retrô, sistema de pontuação por dificuldade e casos investigativos com pistas reais, ruído narrativo e documentos falsos dentro do universo ficcional.

O projeto mistura desenvolvimento web, narrativa interativa, design de interface, investigação, suspense e lógica de jogo.

## Sobre este desafio técnico

Neste desafio técnico, Alexandre criou um assistente simples de IA capaz de decidir entre responder usando um modelo de linguagem ou acionar uma ferramenta externa.

A ferramenta principal é uma calculadora segura, implementada sem uso direto de `eval`, com validação controlada de expressões matemáticas. O projeto também utiliza um LLM para perguntas gerais.

Como diferencial criativo, o projeto pode incluir uma ferramenta de perfil do criador. Essa ferramenta permite que o assistente responda perguntas sobre Alexandre, seus projetos e suas obras com base em documentos locais controlados.

Esse recurso simula, de forma simples, um fluxo de resposta fundamentada em contexto externo, semelhante a um RAG básico, mas sem adicionar complexidade desnecessária ao MVP.
